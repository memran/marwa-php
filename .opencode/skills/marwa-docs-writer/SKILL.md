# Skill: marwa-docs-writer

Goal:
Write or update Marwa documentation clearly and consistently.

Use With:

- marwa-framework
- marwa-module-author

Documentation Rules:

- Explain practical usage first.
- Keep language simple.
- Use real module examples.
- Prefer Users module pattern when relevant.
- Document conventions, not internal implementation noise.
- Update README only when user-facing behavior changes.
- Update AGENTS.md only when repository-wide convention changes.

Write Docs For:

- module usage
- installation
- configuration
- routes
- views
- migrations
- scheduled tasks
- service providers
- testing
- troubleshooting

Avoid:

- over-documenting obvious PHP behavior
- documenting framework internals inside starter docs
- adding stale examples
- inventing APIs not present in code

Required Output:

1. Docs file path
2. Section title
3. Final documentation text
4. Examples
5. Any related files needing update
